## 一、项目背景与使用场景

### 背景

本项目面向农药销售行业，服务于一线销售人员。销售人员在与农户沟通时，需要根据农户描述的情况（种的什么作物、遇到了什么虫/草/病），快速从公司现有产品中找到合适的农药推荐给农户。

目前依赖人工记忆或翻纸质资料，效率低，容易遗漏或推荐错误。

### 目标

建立一套结构化的农药产品知识库，销售人员通过全程点按钮的方式逐步筛选，最终快速获得推荐产品列表及用法说明。后台管理人员可自行维护产品、作物、病虫草害等数据，无需开发人员介入。

### 部署方式

本地局域网部署，单机运行，所有人通过浏览器访问，无需安装任何客户端。端口 9000，局域网内完全开放，无需登录认证。

## 三、"作物"的统一定义（重要）

在本系统中，"作物"的含义做扩展处理：

**作物 = 所有可以作为施药场景的载体**，包括：

- 农作物：玉米、小麦、花生、大豆、高粱、水稻、棉花……
- 非农作物场景：果园、茶园、橡胶园、田埂、路边、铁路沿线、荒山荒地、城市园林……

灭生性除草剂的适用范围（果园/田埂/路边等）在数据库中与农作物一样，均作为 `crops` 表中的一条记录，不做特殊区分。查询流程对所有二级分类统一走"选作物"步骤，无需跳过逻辑。

**好处：**
- 代码逻辑统一，无需对灭生性除草剂做特判
- 管理员可以自由增加新的"作物/场景"，系统自动出现在按钮中
- 灭生性除草剂查询时，用户选"果园"或"田埂"等，逻辑与选"花生"完全一致

## 四、查询流程（按钮式，已确认）

```
第一步：选择一级分类（可多选）
        [除草] [杀虫] [杀菌] [调节剂] [叶面肥] [拌种剂]

第二步：选择二级分类（可多选，显示所有已选一级的子类）
        除草示例：[灭生性除草剂] [封闭除草剂] [玉米除草] [小麦除草] [夏季除草剂单剂]
        当选择多个一级分类时，二级按钮展示所有已选一级的子类合集

第三步：选择作物/场景（动态加载，只显示该分类下实际有产品覆盖的选项）
        封闭除草剂示例：[玉米] [大豆] [花生] [棉花] [大蒜] [洋葱] ...
        灭生性除草剂示例：[果园] [茶园] [田埂] [路边] [荒山荒地] ...

第四步：选择防治对象（动态加载，按一级分类类型过滤）
        规则：根据已选一级分类的类型，只显示对应类型的防治对象
        选择[除草]时 → 仅显示杂草类防治对象
        选择[除草]+[杀虫]时 → 仅显示杂草+虫害类防治对象

第五步：展示匹配产品卡片列表
```

**面包屑导航**：每步选择后顶部显示当前路径，可点击任意节点回退并重新选择。

**顶部筛选卡片**：当前已选条件以芯片(Chip)形式展示在顶部，每个条件可单独点击×移除。

**关键规则**：每步按钮均为动态查询结果，不会出现"选完没有产品"的死路。

## 五、数据模型（6张表）

### 5.1 分类表 `categories`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| parent_id | INTEGER | 0 = 一级分类 |
| name | TEXT | 分类名称 |
| sort_order | INTEGER | 显示排序 |

### 5.2 作物表 `crops`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| name | TEXT | 名称（农作物或非农作物场景均可，如"花生"、"果园"、"田埂"）|
| sort_order | INTEGER | 显示排序 |

### 5.3 防治对象表 `targets`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| name | TEXT | 名称，如"马唐"、"蚜虫"、"白粉病" |
| type | TEXT | weed=杂草 / pest=虫害 / disease=病害 |
| sort_order | INTEGER | 显示排序 |

### 5.4 产品表 `products`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| category_id | INTEGER FK | 关联二级分类 id（无二级则关联一级分类 id）|
| name | TEXT | 含量+名称+剂型，如"33%草甘膦铵盐 水剂" |
| dosage | TEXT | 用量，如"200克/桶水" |
| usage | TEXT | 使用方法及施药时期 |
| notes | TEXT | 特点及注意事项 |
| is_active | INTEGER | 1=上架 / 0=下架，默认1 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 5.5 产品-作物关联 `product_crops`

| 字段 | 类型 | 说明 |
|------|------|------|
| product_id | INTEGER FK | |
| crop_id | INTEGER FK | |

> 复合主键 (product_id, crop_id)

### 5.6 产品-防治对象关联 `product_targets`

| 字段 | 类型 | 说明 |
|------|------|------|
| product_id | INTEGER FK | |
| target_id | INTEGER FK | |

> 复合主键 (product_id, target_id)

## 九、API 设计

### 分类

| Method | Path | 说明 |
|--------|------|------|
| GET | /api/categories | 完整分类树（一级 + 下属二级）|
| POST | /api/categories | 新增分类 |
| PUT | /api/categories/:id | 修改分类 |
| DELETE | /api/categories/:id | 删除分类 |

### 作物

| Method | Path | 说明 |
|--------|------|------|
| GET | /api/crops | 全部作物列表 |
| POST | /api/crops | 新增作物 |
| PUT | /api/crops/:id | 修改作物 |
| DELETE | /api/crops/:id | 删除作物 |

### 防治对象

| Method | Path | 说明 |
|--------|------|------|
| GET | /api/targets | 全部防治对象（可 ?type=weed/pest/disease 过滤）|
| POST | /api/targets | 新增 |
| PUT | /api/targets/:id | 修改 |
| DELETE | /api/targets/:id | 删除 |

### 动态过滤（驱动查询页按钮）

| Method | Path | 参数 | 说明 |
|--------|------|------|------|
| GET | /api/filter/crops | category_ids | 多个分类ID（逗号分隔），该分类下有产品覆盖的作物列表 |
| GET | /api/filter/targets | category_ids, crop_ids | 在多个分类+作物条件下可用的防治对象列表 |

### 产品

| Method | Path | 说明 |
|--------|------|------|
| GET | /api/products | 列表（?category_ids=&crop_ids=&target_ids=&keyword=&is_active= &category_id=）注：category_ids支持多分类，category_id为单分类向后兼容 |
| GET | /api/products/:id | 单品详情（含关联作物和防治对象）|
| GET | /api/products/:id | 单品详情（含关联作物和防治对象）|
| POST | /api/products | 新增（body 含 crop_ids[] + target_ids[]）|
| PUT | /api/products/:id | 编辑（同上）|
| DELETE | /api/products/:id | 删除 |

## 十、页面功能说明

### 查询页（index.html）—— 销售人员日常使用

**交互流程：**
1. 页面顶部显示当前筛选卡片（已选条件以芯片形式展示，可单独移除）
2. 下方显示面包屑导航（当前选择路径，可点击任意节点回退）
3. 主区域显示当前步骤的按钮组（一级分类 → 二级分类 → 作物 → 防治对象）
4. 一级和二级分类支持多选，选择多个一级时二级展示所有对应子类
5. 防治对象步骤根据已选一级分类的类型自动过滤（除草→只显示杂草，杀虫→只显示虫害）
6. 每步选择后，自动请求下一步的动态按钮数据并刷新
7. 最终一步选完后，底部展示产品卡片列表
8. 点击产品卡片展开详情（用量、使用时期、注意事项）
9. 顶部保留搜索框，支持直接输入关键字搜索（跳过按钮流程的备用入口）
10. 页面为单列全宽布局（无侧边栏），筛选项和结果表格占满可用空间

### 管理页（admin.html）—— 数据维护

**四个 Tab：产品 / 作物 / 防治对象 / 分类**

- **产品 Tab**：列表表格 + 新增/编辑弹窗，弹窗内含作物多选、防治对象多选（按杂草/虫害/病害分组）、上架状态
- **作物 Tab**：列表 + 增/改/删
- **防治对象 Tab**：列表（按类型分组显示）+ 增/改/删
- **分类 Tab**：树形展示 + 增/改/删

## 十二、实现约定

1. 所有接口统一返回 JSON，格式：`{"code": 0, "data": ..., "msg": "ok"}`
2. 错误时 `code` 非0，`msg` 说明原因
3. 删除有产品关联的作物/防治对象/分类时，接口返回错误提示，不强制删除
4. 产品 `is_active=0` 时查询页不显示，管理页仍可见
5. 种子数据使用 INSERT OR IGNORE，重复启动不重复写入
6. 所有中文内容统一使用 UTF-8 编码
